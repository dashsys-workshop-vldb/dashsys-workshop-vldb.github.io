Systems Track Overview

In addition to the general research track, DASHSys features a dedicated Real-World Systems Track focused on deployed, production-grade, and demo-ready systems.

Submissions should clearly describe:

  • The system architecture and design principles
  • The real-world problem setting and deployment context
  • Key technical challenges and solutions
  • Evaluation methodology (e.g., performance, cost, reliability, usability)
  • Lessons learned and open challenges

The DASHSys Systems Track focuses on practical systems that support data-centric agents operating over structured databases, APIs, and heterogeneous information sources. Participants design systems that translate natural language questions into structured query plans combining database queries and API calls.

The track emphasizes robust prompt design and system architectures that generalize across models and evolving schemas. Submissions should demonstrate practical solutions for building reliable agent-driven data systems.

Task Description

Input

Component Description
Natural language question User query q
Database schema Tables, columns, types, and relationships
Entity catalog Dimension/metric definitions and valid values
API specification Endpoints, parameters, and response formats

Output

Systems must generate a structured JSON response containing:
Output Component Description
Full LLM prompt System instructions, few-shot examples, schema context, and user query
SQL queries Queries retrieving relevant database information
API calls Specifications to retrieve supplementary data

Participants submit:
  • the full prompt (primary artifact)
  • generated SQL queries
  • generated API calls

SQL/API execution environments are not provided, but participants may simulate them during development.

Evaluation

Systems will be evaluated on:

  • Correctness
  • Completeness
  • Structural validity of generated query plans

Out-of-sample Generalization

A held-out dataset subset will test system generalization to unseen schema elements.

For these cases, organizers will re-execute the submitted prompt using an undisclosed model from the GPT-3.5 → GPT-5 family to regenerate SQL and API calls.

This evaluates prompt robustness across different model capability levels rather than overfitting to a specific model.

Submission Guidelines

System descriptions must follow the VLDB Real-World Systems Track format.

Submissions should include:

  • Ablation studies
  • Benchmark results
